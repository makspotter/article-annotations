# Article Annotator

Небольшое одностраничное приложение для разметки текста статей: создает цветные выделения с заметками, позволяет просматривать и править текст, хранит список статей в браузере.

## Демо
- https://unadesk-test.vercel.app/

## Особенности
- Отслеживание изменений без зон: `provideZonelessChangeDetection()` снижает расходы, уменьшает сборку, делает обновления предсказуемыми.
- Angular Signals: реактивные `signal`, `computed`, `effect` - без лишнего RxJS там, где потоки не требуются.
- Новый синтаксис шаблонов: `@if / @for` вместо `*ngIf / *ngFor` - читается проще, меньше лишних оберток.
- Автономные компоненты и маршруты: приложение без `NgModule`, страницы подгружаются по мере необходимости.
- Умное расположение элементов
  - Подсказка: центровка по якорю, переворот вверх/вниз, ограничение видимой областью, стрелка настраивается CSS-переменными.
  - Окно аннотации: позиционируется относительно выделения, управляет фокусом, блокирует прокрутку подложки. 
- Обработка выделения: DOM-диапазон превращается в символьные индексы строки через `TreeWalker`, убираются лишние пробелы, перекрывающиеся аннотации запрещены.
- Сохранность аннотаций при правке: диапазоны пересчитываются по дельте изменений (префикс/суффикс), поврежденные диапазоны удаляются.
- Адаптивная разметка: мобильное боковое меню с фокус-ловушкой и доступностью (Escape, возврат фокуса).
- Хранение в браузере: `localStorage` с версионированием ключей (`text-annotator:v1:articles`).

## Технологии
- Angular 21, TypeScript 5.9, Standalone Components, Signals, Zoneless.
- ESLint + Prettier для качества кода; Vitest для unit-тестов.

## Требования
- Node.js ≥ 18.19 (рекомендовано 20 LTS)
- npm ≥ 10

## Установка и запуск
1. Установить зависимости:
   - `npm ci` (или `npm install`)
2. Запустить dev-сервер:
   - `npm start`
   - Открыть http://localhost:4200

## Сборка и деплой
- Продакшен-сборка:
  - `npm run build`
  - Результат в `dist/text-annotator` - статические файлы для любого статического хостинга (Vercel/Netlify/Nginx и т.д.).
- В дев-конфигурации (sourcemap, без оптимизаций):
  - `npm run watch`

## Скрипты npm
- `npm start` - dev-сервер.
- `npm run build` - продакшен-сборка.
- `npm run watch` - dev-сборка с вотчером.
- `npm test` - юнит‑тесты (Vitest).
- `npm run lint` - ESLint проверка.
- `npm run lint:fix` - ESLint с автофиксами.
- `npm run format` - форматирование Prettier.
- `npm run format:check` - проверка форматирования.

## Хранение данных
- Все статьи и аннотации сохраняются в браузерном `localStorage` под ключом `text-annotator:v1:articles`.
- Очистка данных возможна через DevTools → Application → Local Storage.

## Ограничения и заметки
- Приложение рассчитано на запуск в браузере (есть обращения к `window/document`), SSR не предполагается.
- Для очень больших текстов может потребоваться виртуализация - текущая реализация хорошо работает на типичных размерах статей.
